# Module 10 - Git

The hands-on commands are completed in `GitCommands.md`.

## Objective
Learn how to use Git for version control, collaboration, and change tracking in a team environment.

## What you practice
- Initializing a repository and creating the first commit
- Configuring user identity and default editor settings
- Using `.gitignore` to exclude generated files and logs
- Creating and switching branches
- Merging branches and resolving merge conflicts
- Pulling from and pushing to a remote repository

## Key concepts
- Repository, commit, branch, and merge
- Working tree vs staging area
- Remote tracking branches
- Conflict resolution workflow
- Common Git commands for daily development

## Hands-on flow
1. Create a new repository.
2. Add a file and commit it.
3. Create a branch and add changes there.
4. Merge the branch back and resolve any conflicts.
5. Push the final result to a remote repository.
